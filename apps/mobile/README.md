# @trailme/mobile — TrailMe field-guard app

The Expo (SDK 52) / React Native field app guards run on patrol: it background-tracks GPS, shows same-site peers on a Mapbox map with fading trails, and lets a guard clock in/out and tag inspection checkpoints. **This is a custom dev-client app — it can NEVER run in Expo Go**, because it links two native modules (`react-native-background-geolocation` and `@rnmapbox/maps`) that Expo Go does not bundle. Build a dev client with EAS (`eas build --profile development`), install it on a real device, then run `pnpm --filter @trailme/mobile start` (which launches `expo start --dev-client`). Background GPS and native maps are impossible to validate in a simulator or in Expo Go, so a physical device + dev client is the only supported workflow from day one.

**React is pinned to 18.3.1, not 19** — React Native 0.76 ships against React 18.3.x, and Expo SDK 52 expects that pairing; React 19 is for the Next.js dashboard, not RN. **The transistorsoft license (`RN_BG_GEO_LICENSE`) is a PAID dependency needed at M2** — it is enforced only in RELEASE builds, so dev-client/debug builds run and track fine without it but log an "invalid license" warning; CI must assert a valid license in a release build before launch. Runtime config comes from `EXPO_PUBLIC_SUPABASE_URL` / `EXPO_PUBLIC_SUPABASE_ANON_KEY` / `EXPO_PUBLIC_MAPBOX_TOKEN` (copy `.env.example` to `.env`); native build secrets `MAPBOX_DOWNLOADS_TOKEN` (private Mapbox SDK download) and `RN_BG_GEO_LICENSE` are read by `app.config.ts` at prebuild/EAS-build time and should live in EAS Secrets, never in git. M0 ships the auth + navigation spine and the native config (permissions, Info.plist usage strings, Android 14 `foregroundServiceType=location`); the real tracking/ingest and live-map layers are wired in M2–M4 — see `docs/ARCHITECTURE.md`.
