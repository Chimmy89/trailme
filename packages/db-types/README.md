# @trailme/db-types

The single source of truth for the Supabase Postgres schema as a TypeScript `Database` type, consumed by `@trailme/supabase-client` (and through it, both apps) to give every query end-to-end type safety. Until the first migration lands in M1 there is nothing to generate from, so the package ships a minimal placeholder `Database` (empty `Tables`/`Views`/`Functions`/`Enums`) that satisfies `createClient<Database>()` and keeps the workspace compiling. After any schema change, regenerate the real types with `supabase gen types typescript --linked > packages/db-types/src/database.types.ts`, then repoint the re-export in `src/index.ts` from `./placeholder` to `./database.types` (the placeholder file is deleted at that point). The generated file is committed so consumers never need a live database to type-check.
