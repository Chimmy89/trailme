# @trailme/shared

The cross-cutting contract package: the one place where wire shapes and closed enumerations are defined so the dashboard, the Expo app, and the Supabase edge functions all agree byte-for-byte. It exports the closed enums as `const` tuples paired with their derived literal-union types (`ROLES`/`Role`, `TRACKING_MODES`/`TrackingMode`, `RETENTION_DAYS`, `TIME_WINDOWS`, `LAWFUL_BASES`), the `channelName(siteId)` helper that builds the private per-site realtime channel `site:{siteId}`, zod schemas plus inferred types for the breadcrumb DTO and its batch/ingest-result envelopes, the consolidated `positions` broadcast payload (server-stamped identity, advisory live ticks), the `~24h` device-token claim shape (`org_id`/`guard_id`/`site_ids`/`kind:'device'`), and small runtime guards (`isRole`, `isTimeWindow`, `assertNever`, …). It is pure TypeScript with a single dependency on `zod` — no Supabase, Mapbox, React, or Node APIs — so it imports cleanly into every runtime in the monorepo. Validate at every trust boundary by parsing with these schemas; consume the inferred types everywhere else.
