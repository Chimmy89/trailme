# TrailMe — Supabase backend

The backend-as-code for TrailMe: an EU-region Supabase project (Postgres 15 + PostGIS + Realtime + Auth + RLS) that owns multi-tenant identity, authorization, and (from M1) the breadcrumb firehose. **M0** lands the tenancy spine only: ordered migrations create the PostGIS/pg_cron/UUIDv7 extensions and a private `authz` schema (`0001`), the tenant tables `organizations` / `org_settings` (with a trigger forbidding `always_on` until a DPIA is filed) / `sites` / `profiles` / `memberships` (one-org-per-user) / `guard_disclosures` (`0002`), `FORCE` Row Level Security with `authz.org_id()` / `authz.has_role()` / `authz.has_site_access()` SECURITY DEFINER helpers doing **live** membership lookups so a cross-org `SELECT` returns zero rows (`0003`), and the NULL-safe, fail-closed `custom_access_token_hook` that stamps `app_metadata.org_id/role/site_ids` from the membership row and never elevates role (`0004`). `seed.sql` provisions one org, a shift-gated/30-day settings row, one site, and two login-ready users (org_admin + guard) with a disclosure, so the M0 login test has deterministic data. The three Deno edge functions under `functions/` (`ingest-breadcrumbs`, `mint-device-token`, `relay-positions`) are typed **stubs returning 501** — their M1/M3 responsibilities are documented in each file header. Run locally with `supabase start` then `supabase db reset` (applies migrations in filename order, then seeds); the EU region is pinned at project-create time per `config.toml`'s residency note.
